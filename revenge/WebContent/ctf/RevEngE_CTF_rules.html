Introduction


The RevEngE system hosts binary reverse engineering challenges. Our goal is to study humans as they complete these challenges and we hope that they will provide new insights into code obfuscation and reversing practices.


While the main RevEngE system hosts ongoing difficult problems, we also have periodic CTF-style events with graduated problem difficulties. Subjects can participate in these events and compete with other reverse engineers. As an incentive, we provide cash prizes for the teams with the top scores!


Below are the rules.



Quickstart


To get right to participating, do this:


  • Create a new virtual machine in VirtualBox or VMWare. Give it plenty (>=2 processors, >=4GB memory) of resources, since some CPU/memory will be used by the monitoring software.
  • Install the OS. We support Kali and Ubuntu.
  • Boot the OS.
  • Open a browser, and go to http://revenge.cs.arizona.edu/RevEngECompetition/ctf/RevEngE_CTF_signup.html and create an account and log in.
  • Download the file setup.sh
  • Run
    • > chmod a+rx setup.sh
    • > sudo ./setup.sh
  • Review the consent agreement:
    • revenge.cs.arizona.edu/RevEngECompetition/Professionals.pdf
  • Give consent or exit. If you give consent, the monitoring software will be installed.
  • Download the challenges (such as CTF_1.00) from the challenge page on the website.
  • Solve the challenge, i.e. turn CTF_1.00 into deobfuscated C, “CTF_1.00.c”.
  • Upload the solution “CT_1.00.c”. and the corresponding trace file to challenge page on the website.


Who may participate?


Anyone who would like to may participate by signing up on the RevEngE web application. Participants may be individuals or teams. All requirements for individual participants apply equally to each individual on a participating team.



Prizes


There will be a cash prize for each event to the top placing team of no more than $10,000 per event. For certain events, there may be a prize for the best cumulative performance throughout those events, also not to exceed $10,000.



Scoring


Each challenge has an associated score representing the difficulty level we anticipate for that challenge. A subject’s score for solving a challenge is either 100% of the challenge in the case of right/wrong submissions (such as is the case for removing a password check) or a percent of the challenge score based on how well the solution performs in testing. The type of scoring will be listed per event and per challenge as appropriate.


Unless otherwise noted, scoring for all problems is as follows:


  • When you upload your submission, it is stored in the RevEngE Competition database.
  • If the submission is a source code submission (ie, you submitted a .c file) then it is compiled and run with random cases and/or cases developed for that particular challenge.
  • The results are compared to the original (non obfuscated) file to determine if they are likely functionally correct.
  • Your submission is compared to the original (non obfuscated) file in the number of instructions executed in running these cases.
  • The instructions counted are high level, LLVM instructions to ensure independence from operating system and hardware variables.
  • If the submission executes functionally identically to the original according to our tests and operates within a performance threshold (generally within a factor of 2 instructions, but could vary depending on challenge) of the original, then it is considered possibly correct.
  • If a submission is possibly correct, then we will manually evaluate it for credit following the close of the competition. This evaluation includes, but is not limited to, control flow comparisons and manual cases.


Challenge Progression


All unlocked challenges will be available to all participants at the given time. The researchers operating RevEngE may release new challenges as they see fit. Additionally, solving some challenges may make new challenges visible to everyone.


All other rules of the RevEngE system, particularly those related to data collection, also apply unless otherwise stated. Prizes can only be won if the data collection system is installed and working on all devices and you submit the data it collects. These rules may be found on the RevEngE Competition site. In order to ensure fairness, the maintainers of RevEngE reserve the right to extend or end a competition at any time, particularly in response to attempts to hack the RevEngE system.



Instructions


In order to participate, you will first need to make an account here. This account will be used to keep track of your progress for every CTF event we host. Additionally, past CTF event problems will be available for you to keep hacking at on the main Revenge Competition Engine after the event ends.


Once you sign up for an account, you may login to the CTF engine by clicking the Challenges link. Once logged in, your problems will appear. If they do not, the system may be generating copies of the problems for you, in which case refreshing after a minute should make more problems appear. Click on a problem to see what you need to do!



Data Collection


RevEngE utilizes a custom data collection software to monitor users as they solve challenges. This generates a data set that we intend to mine to derive and evaluate modern reverse engineering methods. We highly recommend installing the software on a virtual device so that we do not see things like entering passwords on sensitive websites. The software collects these pieces of data:


  • Screenshots
  • Mouse and keyboard input
  • Running processes
  • Active window information

Once you have an account, follow these steps use the data collection software:


  • Download and configure your Debian variant (we test using Kali and Ubuntu, and these are the versions we support) virtual machine
  • Once your VM is running, download and run with admin/sudo/root the setup.sh script, located on the Challenges page
  • With the data collection running, download and solve challenges
  • Upload your challenge submissions, and when you do check the "Upload Collected Data" to submit your data (because this takes a while, you may want to do this only for your final submission)


Prize Eligibility


The prize of a competition is awarded to the highest scoring participant(s). In order to be eligible for a prize, a participant must submit at least one correct solution to a challenge. A successful solution is one which fulfills the challenge requirements and is completed through the use of reverse engineering techniques applied only to the challenge's files. This process must be recorded using the data collection software and that data must be submitted with the solution in order for it to be successful. Note that solutions obtained through compromising the RevEngE systems are not considered successful.


Binary reverse engineering challenges contain a "challenge.exe" file, which is an ELF x86 executable that starts out as source code "original.c" generated by Tigress' Random Hash Function generator (see https://tigress.wtf/randomFuns.html), then obfuscated with a sequence of Tigress' obfuscating transformations, compiled with clang or gcc, and stripped of symbols. Your task is to turn "challenge.exe" back into "solution.c", a deobfuscated version of "challenge.exe", in idiomatic C, and as close to "original.c" as possible. To be successful and claim a prize, your "solution.c" should be non-trivial and "reasonably close" to "original.c". This is evaluated using the grading methodology outlined above.
